## Context

Mounted travel only makes sense in the open world. Entering battles, named-map interiors, or dungeons triggers automatic dismount. Returning to the overworld restores the mount state. Visually, the mount is distinct via a mount sprite and dust particles while moving.

## Research Notes

- **Auto-dismount triggers:**
  - **Battle start:** Listen to **`GameBus.enemy_engaged`** signal (cite **`autoloads/GameBus.gd`** for signal definitions; search for "enemy_engaged" in the codebase). When this fires, call `SaveManager.dismount()` (new method from TID-179). Store `was_mounted = SaveManager.is_mounted` before dismounting so we can restore it on return.
  - **Named map entry:** Listen to **`SceneManager`** map-change event or hook. Check if the new map is not "main" (cite **`docs/agent/ui-and-scene-management.md`** for SceneManager scene stack). When entering any named map (interiors, dungeons, madrian), call `SaveManager.dismount()` and store `was_mounted`.
  - **Dungeon entry:** Same as named maps — if current_map != "main", dismount. Dungeons are named maps in the stack.
  - **Return to overworld:** When returning to "main" map (checked via `SaveManager.current_map` or a signal), restore: if `was_mounted` was true, call `SaveManager.summon_mount(SaveManager.active_mount)`. This auto-remounts without player action.

- **Mount state machine (state is stored in SaveManager):**
  - `is_mounted: bool` — true only in overworld and actively riding. Set to false on dismount, restored on return.
  - `active_mount: String` — the owned mount ID. Persists across battles/interiors.
  - Simplest: Just toggle `is_mounted` flag. Speed multiplier and visuals key off this flag.

- **Mounted sprite visual:**
  - **Player sprite setup (cite `scenes/world/entities/Player.gd` lines 32–48):** Currently a single `Sprite3D` at position `Vector3(0.0, frame_h * 0.5, 0.0)` where `frame_h = _WalkTex1.get_height() * PIXEL_SIZE`. The sprite is billboard-enabled, centered at the origin, and lifted above y=0 so the feet don't clip into the floor (cite CLAUDE.md "Sprite3D: Depth Clipping Into Floor" section, formula `sprite.position.y = pixel_height * pixel_size * 0.5 + small_margin`).
  - **Mount sprite creation:** Add a second `Sprite3D` child to the Player node, positioned **below** the player sprite (lower y value). Use a simple pixel-art horse/mount texture generated by TextureGen or hand-authored. Suggest: `assets/textures/pixel_art/mount_horse_walk.png` (similar dimensions to player, ~32–48 px wide, to match scale).
    - Position mount at `Vector3(0.0, frame_h_player * 0.25, 0.0)` (roughly half the player height, so it sits under the player visually).
    - Only visible when `SaveManager.is_mounted == true`.
    - Sync animation frame with player frame (use the same `_anim_frame` counter).
  - **Texture generation:** If the project uses TextureGen (cite **`game_logic/TextureGen.gd`** if it exists), generate the mount sprite dynamically. Otherwise, create a static PNG asset.

- **Dust particle effect:**
  - Add a `GPUParticles3D` node to the Player node when mounted and moving (similar to grass particle usage elsewhere).
  - Cite existing particle usage in the codebase (search `GPUParticles3D` in scenes/ and game_logic/). GID-023 (game feel polish) may have particle examples.
  - Particle behavior: small dust/dirt particles emitted from the player's feet, moving outward and dissipating. Only emit while `_is_moving == true` (cite Player.gd line 88 for the movement check). Disable particle emission when idle or dismounted.
  - Material: simple additive or transparent material, brownish/gray color for dust.

- **Jump while mounted:** Design decision: **Allow jumping while mounted** (same physics as unmounted). No special behavior needed. The player can jump off obstacles while riding.

- **Dismount/remount state machine:**
  - Battle start → dismount (set `is_mounted = false`), store `was_mounted = true` in a local/persistent variable.
  - Battle end → if `was_mounted`, remount (set `is_mounted = true`).
  - Named map entry → dismount, store `was_mounted`.
  - Return to main → if `was_mounted`, remount.
  - Simplest: SaveManager tracks `was_mounted_before_battle: bool`. On `GameBus.enemy_engaged`, set it. On battle end (listen to `GameBus.battle_won` or similar), check it and remount if true.

- **Signal integration:**
  - **`GameBus.enemy_engaged(enemy_data)`** — dismount on receipt. Cite signal definitions if available.
  - **`GameBus.battle_won`** or **`GameBus.battle_lost`** — remount based on `was_mounted` flag.
  - **Scene/map entry signals** — listen to SceneManager or a custom signal when map changes. Cite **`docs/agent/ui-and-scene-management.md`** for how map transitions are signaled.

- **Tests:** Headless test file `tests/test_mount_dismount_visuals.gd`:
  - Test auto-dismount: Player mounted in "main", simulate `GameBus.enemy_engaged` event, verify `is_mounted` becomes false and speed reverts to SPEED.
  - Test remount on battle end: After battle, verify `is_mounted` is restored to true if `was_mounted` was set.
  - Test interior dismount: Simulate map transition to a named map (e.g. "madrian"), verify dismount. Transition back to "main", verify remount.
  - Test particle emission: mounted + moving → particles emitted. Idle or dismounted → no particles.
  - Test sprite visibility: mounted → mount sprite visible, player sprite raised. Dismounted → player sprite at normal height, mount sprite hidden.
