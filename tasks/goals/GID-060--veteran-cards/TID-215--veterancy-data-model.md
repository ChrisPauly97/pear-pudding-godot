## Context

GID-060 extends GID-028's per-instance card model into battle memory: per-instance counters `kills` and `battles_survived`, persisted in the save, plus rank thresholds (e.g. rank 1 at 5 kills or 10 battles, rank 2 at 15/25, rank 3 at 40/60 ‚Äî exact numbers decided in Plan phase) that grant small stat bumps (+1 HP or +1 ATK per rank) and an earned title suffix, and a `custom_name` for player renaming. This task lays the data foundation: instance fields, save migration, and a pure veterancy-math helper that TID-216 (battle attribution) and TID-217 (UI) build on.

**Persistence format:**
- `save()` (line 411) writes `"owned_cards": owned_cards` directly into the JSON dict ‚Äî new keys added to instance dicts persist automatically with no serializer change.
- `load_save()` (line 351) does `owned_cards.assign(data.get("owned_cards", []))` after `_apply_migrations(data)`.
- `CURRENT_SAVE_VERSION: int = 14` (line 184). Migration pattern: one `static func _migrate_vN_to_vN+1(data: Dictionary) -> void` per bump, chained in `_apply_migrations()` (line 320). This task should bump to 15 and add `_migrate_v14_to_v15` backfilling `kills: 0`, `battles_survived: 0`, `custom_name: ""` on every dict in `data["owned_cards"]`.
- `new_game()` (line 128) builds the starter collection via `add_card_instance(tid, "common")` ‚Äî new fields must be initialised there too (inside `add_card_instance`).
- The v9‚Üív10 migration (line 254) is the reference example for migrating `owned_cards` entry shape.

**Where rank/title math should live:**
- Follow the `CardDropUtil` pattern: `/home/user/pear-pudding-godot/game_logic/CardDropUtil.gd` is a static-func utility with **no class_name** ("callers preload this file directly", per its header comment and the CLAUDE.md class_name rule). Create e.g. `game_logic/VeterancyUtil.gd` the same way: static funcs `rank_for(kills: int, battles_survived: int) -> int`, `title_for(rank: int) -> String` (or per-template title tables), `stat_bonus_for(rank) -> Dictionary`, `display_name(inst: Dictionary, base_name: String) -> String`.
- Threshold constants: canonical-constants rule says gameplay constants live in `/home/user/pear-pudding-godot/autoloads/IsoConst.gd` ‚Äî `RARITY_CONFIG` (line 48) and `RARITY_ORDER` (line 54) are the precedent; add e.g. `VETERANCY_RANKS` there (kills threshold, battles threshold, hp/atk bump, title per rank). Rank rule is OR-based: rank N when `kills >= kill_threshold[N]` **or** `battles_survived >= battle_threshold[N]`.
- Display-name precedence to settle in Plan: `custom_name` if non-empty, else `base_name + " the " + title` when rank ‚â• 1, else base name.

**Card template names** come from `CardRegistry.get_template(id)` (autoload, preloaded as `const CardRegistry = preload("res://autoloads/CardRegistry.gd")` in SaveManager line 4); template dict keys seen in use: `name`, `attack`, `health`, `cost`, `card_class`.
